Lesson 3: Understanding the Structure of a C++ Program

Let's take a closer look at what makes a C++ program work. Every part of the code has a purpose!

📦 The Core Components

  • #include <iostream> – Adds input/output functionality.
  • #include <string> – (Optional) Adds support for strings.
  • using namespace std; – Lets you avoid typing std:: everywhere.
  • int main() { ... } – The starting point of every C++ program.
  • return 0; – Tells the system that the program ended successfully.

📋 Full Program Structure Example

    #include <iostream>       // Required for cout and cin
        #include <string>         // Required for using strings
    
        using namespace std;        // Avoids writing std:: before cout, cin, etc.
    
        int main() {                // Main function: entry point of the program
            string name;            // Variable to store name
            int age;                // Variable to store age
    
            cout << "Enter your name: ";  // Output to user
            cin >> name;                  // Input from user
    
            cout << "Enter your age: ";
            cin >> age;
    
            cout << "Hello, " << name << "! You are " << age << " years old." << endl;
    
            return 0;               // End the program
        }

🔍 Breakdown of Keywords

  • #include – Think of it like "copy-pasting" code from a library into your program.
  • <iostream> – It is a library; Contains input/output tools like cin and cout.
  • using namespace std; – Means "I'm using the standard tools, so I won’t type std:: every time."
  • int main() – The function that runs first when you launch your app.
  • return 0; – The signal for "everything ran successfully."

📚 What Are Libraries?

Libraries are prewritten code that give your program superpowers. You don't have to build everything from scratch! Want input/output? Use <iostream>. Want strings? Use <string>.
